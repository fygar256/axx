# axx マクロ層 構文リファレンス

アセンブラ本体に渡す前に走るソース→ソース変換段。ラベル値・`.equ`・`$`/`$$` は
参照できない（relaxation の各パスで展開結果が変わらないことを保証するため）。

axx.py と caxx.c の両方に同一仕様で実装されている。両者の唯一の差は数値の
表現で、Python 版は多倍長整数、C 版は int64。マクロ時演算が 64bit を超える
場合のみ結果が食い違う（マクロ層が出力するのは常にソーステキストなので、
アセンブラ本体の 256bit 式評価には影響しない）。

## 文

すべて行頭（先頭の空白を除く）の `!` から始まる。

| 構文 | 意味 |
|---|---|
| `!def name(p1, p2, p3 = 既定値) { ... }` | マクロ/コンパイル時関数の定義 |
| `!return expr` | 返り値。本体からの早期脱出も兼ねる |
| `!if expr !then { ... } !elif expr !then { ... } !else { ... }` | 条件分岐 |
| `!while expr { ... }` | ループ |
| `!break` / `!continue` | ループ制御 |
| `!set name = expr` | 既存スコープを内側から探して代入、なければ現スコープに作成 |
| `!local name [= expr]` | 現スコープに宣言 |
| `!undef name` | 変数/マクロの削除 |
| `!name(a, b)` | マクロを文として展開 |
| `!include "file"` | マクロ時テキスト取り込み |
| `!error expr` | 展開を中止しエラー |
| `!warning expr` / `!echo expr` | stderr へ出力 |

`{` は必ずヘッダ行の末尾、閉じ `}` は行頭。`; コメント` は文の後ろに書ける。

## 式の中への埋め込み

| 記法 | 意味 |
|---|---|
| `!{expr}` | 値をテキストに展開 |
| `!{expr:04x}` | Python の書式指定を適用 |
| `\!{` | リテラルの `!{` |

## 値と演算子

値は整数と文字列のみ。演算子は C 準拠で
`?:` `\|\|` `&&` `\|` `^` `&` `==` `!=` `<` `<=` `>` `>=` `<<` `>>` `+` `-` `*` `/` `%`
と単項 `-` `+` `~` `!`。`/` と `%` は C と同じ 0 方向切り捨て（`-7/2 == -3`）。
`+` はどちらかが文字列なら連結、`"ab" * 3` は繰り返し。
整数リテラルは `10` / `0x1f` / `0b1010` / `0o17`、`_` 区切り可。
`'A'` は 1 文字なら文字コード、複数文字なら文字列。

## 組み込み関数

`len(s)` `hex(v[,桁])` `str(v)` `int(s[,基数])` `upper(s)` `lower(s)`
`substr(s,開始[,長さ])` `abs(v)` `min(...)` `max(...)` `uid()` `defined(name)`

## マクロ内の暗黙変数

| 名前 | 内容 |
|---|---|
| `__id__` | 呼び出しごとに一意な整数（ローカルラベル生成用） |
| `__name__` | マクロ名 |

## CLI

| オプション | 意味 |
|---|---|
| `--no-macro` | マクロ層を完全に無効化 |
| `-P [FILE]`, `--macro-expand [FILE]` | 展開結果だけを出力して終了（省略時は標準出力） |

caxx.c 側も同じオプション名。C 版の `-P` はファイル名を省略できるようにする都合上、
パターンファイルとソースファイルの両方を指定した後ろに置いたときだけ次の引数を
出力先として解釈する（例: `caxx pat.axx src.s -P out.s`）。

## 後方互換の扱い

- `!` 行を横取りするのは、キーワード・定義済みマクロ名・直後が `(` の場合のみ。
  VLIW の `!!` および `!F` / `!D` / `!Q` は一切触らない。
- 行頭 `}` を閉じ括弧として扱うのはブロックが開いているときだけ。
- マクロを含まないソースの出力は従来と完全に一致する。

## 制限

- アセンブラ側の `.INCLUDE` はマクロ層を通らないので、マクロ定義ファイルの
  取り込みには `!include` を使う。
- `!{a ? b : c:04x}`（三項演算子と書式指定の併用）は現状パースできない。
- 対話モード（ソースファイル省略時）はマクロ層を通らない。

## 暴走ガード

再帰 200 段、`!while` 1,000,000 回、生成行 2,000,000 行、`!include` 深さ 64。
いずれも超過でエラー。エラー発生後は以降のパスで展開を打ち切る。

## 例

```asm
!include "lib.inc"

!def table(name, from, to) {
!{name}:
    !set v = from
    !while v <= to {
        DB !{v}
        !set v = v + 1
    }
}

!def loopblk(n) {
    !if n == 0 !then {
        !return
    }
L!{__id__}_top:
        LD A,!{n}
        LD HL,L!{__id__}_top
}

!def sq(x) {
    !return x * x
}

start:
        LD HL,end
!table("mytab", 1, 5)
!loopblk(2)
        LD HL,!{sq(16)}
end:
        NOP
```
